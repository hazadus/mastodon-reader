💬 Day 2/75: Go built-in data structures

Today's note was about using Go slices, maps, arrays, and structs deliberately instead of treating them like black boxes. I kept coming back to the same checks: pick slices, arrays, maps, or structs based on the exact lookup and update behavior you need, be explicit about mutation versus copy operations before the loop gets complicated, and use built-ins because they match the invariant, not just because they are available.

The failure mode worth watching is picking a built-in structure by habit instead of by required behavior. If that happens, the implementation usually looks busy while the invariant is already gone.
