**🕒 20:46 👤 @itamarst@hachyderm.io**
💬 One issue with my library for estimating computational complexity of Rust functions is that it works by finding the model that best fits from a set of known models.

What if the best fit is not the actual complexity model?

Let's say that real complexity is 2^N, but fastest growing known model is N^3. So N^3 is best fit. Oops.

I came up with a scheme to detect this: come up with a new measure of complexity, the ratio of measured real complexity (e.g. CPU instructions) to the model's estimated complexity. Now fit _that_ to the models and find the best one.

If the model is a good match, the best fitting model will be constant.

If the model grows much slower than the real function's complexity does, the ratio will get fit to some growing model... which would indicate it's not an actually good fit.

Coming soon to [https://crates.io/crates/bigoish](https://crates.io/crates/bigoish)
