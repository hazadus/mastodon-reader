ğŸ’¬ In go you can append items to a [slice](https://go.dev/blog/slices-intro) and it will modify the underlying array.

// an array
a := [4]int{1, 2, 3, 4}
// a slice (with a as underlying array)
s := a[:2]

// Modifies underlying array but not s
_ = append(s, []int{5, 6}...)

// [1 2 5 6]
fmt.Println(a)
See it for yourself: [https://go.dev/play/p/rv378Nt_eTB](https://go.dev/play/p/rv378Nt_eTB)

So if you create a slice, pass it to a func , where it is used as the first arg to an append, the chances are that the underlying array is also modified. In turn, if two slices share the same array, modifying one (e. g. with append) can influence the other:

func f(s []int) {
    _ = append(s, []int{5, 6}...)
}

func main() {
    a := [4]int{1, 2, 3, 4}
    s1, s2 := a[:2], a[2:]

    f(s1)

    fmt.Println(s2) // [5 6]
}
