💬 Day 17/75: Binary search in Go

Today's note was about protecting the search interval in binary search in go so every bound update stays justified. I kept coming back to the same checks: state what the active interval still allows before the first loop iteration, move left and right only when the predicate proves one side is impossible, and test the smallest failing boundary because off-by-one errors hide there first.

The failure mode worth watching is moving left and right without a precise definition of what they still allow. If that happens, the implementation usually looks busy while the invariant is already gone.
